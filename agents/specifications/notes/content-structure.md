# 内容结构规范

本文件定义笔记正文中确定性可判定的结构规则。

## 总体结构

每个笔记文件由两部分组成：

1. **Frontmatter**：位于文件开头，用 `---` 定界符包裹的 YAML 块。具体字段定义见 `frontmatter.md`。
2. **主体内容**：紧跟在 frontmatter 之后，使用嵌套无序列表组织的知识内容。

```text
---
title: 中文标题
weight: N00
math: true
---

- **概念A**
    - ...
- **概念B**
    - ...
```

frontmatter 与主体内容之间无空行。主体内容直接以顶层列表条目开始。

## 列表层级

笔记正文使用嵌套无序列表（`-`）组织内容，每级缩进 4 个空格。层级深度不固定，根据知识点的复杂度可以有更多或更少的层次。

```markdown
- **概念**
    - 解释
    - 解释
        - 附加解释
    - **子概念**
        - 解释
```

### 判定规则

- 缩进必须为 4 的倍数（4, 8, 12, ...）
- 不允许混用制表符和空格缩进
- 列表标记统一使用 `-`，不使用 `*` 或 `+`

## 加粗规则

加粗（`**...**`）用于标记术语或概念名称，表示在知识层级中逐渐细化的概念节点。

不加粗的情况：

- 术语的解释内容
- 数学公式
- 描述性文字

```markdown
- **极限**
    - **定义**
        - 设数列 $\{a_n\}$ ...
    - **性质**
        - 唯一性
            - 如果极限存在，则极限唯一。
```

### 判定规则

- 加粗内容应为词或短语，不应对整段解释性文字加粗

## 标题使用

笔记正文中**不使用** Markdown 标题（`##`, `###` 等）来组织内容层级。所有内容层级通过列表嵌套表达。

### 判定规则

- 正文中出现 `##`, `###`, `####` 等标题标记视为不合规

## 顶层条目格式

每个顶层条目表示一个核心概念，格式为：

```markdown
- **概念名**
```

## 锚点放置

`<span id="..."></span>` 仅放置在需要被其他笔记链接的条目行末：

```markdown
- **电场强度** <span id="dkqgis"></span>
```

### 判定规则

- `<span id>` 不出现在行首或单独成行
- `<span id>` 不出现在 frontmatter 中

## 合规判定

以下情况判定为不合规：

- 缩进不是 4 的倍数
- 混用制表符和空格
- 正文中使用了 Markdown 标题（`##` 等）
- 列表标记未统一使用 `-`
- `<span id>` 单独成行
