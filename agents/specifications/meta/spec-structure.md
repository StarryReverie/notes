# Specification 文件结构规范

本文件定义 `agents/specifications/` 目录下所有文件必须遵守的内容结构规则。

## 必须包含的章节

### 标题

文件的第一个内容是 `#` 级标题，描述规范名称。

### 正文

规范的详细描述，按逻辑分节（`##`、`###`）。各节按需划分。

### 合规判定

必须包含 `## 合规判定` 章节，列出明确的判定条件。格式为：

```markdown
## 合规判定

以下情况判定为不合规：

- 具体条件 1
- 具体条件 2
```

## 代码块使用

代码块只包含示例内容本身。描述性标签（如"正确"、"错误"）放在代码块外部，不放在代码块内部。

如需在代码块内标注，使用该语言的注释语法（如 markdown 中无标准注释，应避免标注）。

## 合规判定

以下情况判定为不合规：

- specification 文件缺少 `## 合规判定` 章节
- 代码块内包含非示例内容的描述性文字
