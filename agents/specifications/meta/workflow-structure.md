# Workflow 文件结构规范

本文件定义 `agents/workflows/` 目录下所有文件必须遵守的内容结构规则。

## 必须包含的章节

### 标题

格式为 `# 工作流：名称`。

### 触发条件

必须包含 `## 触发条件` 章节，描述什么情况下使用该工作流。

### 步骤

必须包含 `## 步骤` 章节，使用编号子步骤（`### 1. xxx`、`### 2. xxx`）描述操作序列。

### 参考规范

必须包含 `## 参考规范` 章节，列出该工作流需要参考的 specification 文件。

## 代码块使用

代码块只包含示例内容本身。描述性标签（如"正确"、"错误"）放在代码块外部，不放在代码块内部。

如需在代码块内标注，使用该语言的注释语法（如 markdown 中无标准注释，应避免标注）。

## 合规判定

以下情况判定为不合规：

- workflow 文件缺少 `## 触发条件`、`## 步骤`、`## 参考规范` 中的任一章节
- workflow 步骤未使用编号子步骤
- 代码块内包含非示例内容的描述性文字
